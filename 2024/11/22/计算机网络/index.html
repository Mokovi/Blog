计算机网络概述

计算机网络的分类

  1. 按覆盖范围分类：
    • 个域网（PAN）：覆盖范围一般在10米半径以内，在便携式消费电器与通信设备之间进行短距离通信的网络，如蓝牙耳机等
    • 局域网（LAN）：覆盖范围小，通常局限于单一建筑物或校园，如以太网。
    • 城域网（MAN）：覆盖范围较大，通常跨越城市范围，例如城域网光纤网络。
    • 广域网（WAN）：覆盖范围最广，连接不同城市甚至国家的网络，如互联网。
  2. 按拓扑结构分类：
    • 总线型：所有节点通过共享一条总线连接，常用于早期以太网。
    • 星型：所有设备通过交换机或集线器连接，常见于现代网络。
    • 环型：节点依次连接成闭环结构，如FDDI。
    • 网状型：节点间通过多条链路连接，具备高可靠性和冗余性。
  3. 按使用者分类：
    • 公用网络：供公众使用的网络，如4G/5G网络。
    • 专用网络：为特定组织或企业服务的网络，如企业内部网络（Intranet）。
  4. 按传输方式分类：
    • 电路交换网络：如传统电话网络。
    • 分组交换网络：如互联网，数据以分组形式传输。

计算机网络的层次

  1. OSI参考模型（七层结构）：
    • 应用层、表示层、会话层、传输层、网络层、数据链路层、物理层。
    • 每层完成特定的功能，从高层的用户服务到低层的硬件传输。
  2. TCP/IP模型（四层结构）：
    • 应用层、传输层、网络层、网络接口层。
    • 是互联网的基础模型，实际应用广泛。

  1. 实际应用中的三层模型：

    • 接入层：实现终端接入。

    • 汇聚层：实现局部数据处理与转发。

    • 核心层：实现高性能的数据交换。

计算机网络性能指标

  1. 带宽：单位时间内最大传输数据量，通常以bps表示。
  2. 延迟：数据从源到目的地的时间，分为传播延迟、传输延迟、排队延迟和处理延迟。
  3. 往返时间RTT：数据报文在端到端通信中的来回一次的时间
  4. 吞吐量：单位时间内成功传输的数据量，是实际带宽的体现。
  5. 丢包率：传输过程中丢失数据分组的比例，影响网络可靠性。

互联网的构成

网络边缘：位于互联网边缘与互联网相连的计算机和其他设备,如桌面计算机、移动计算机、服务器、其他智能终端设备
网络核心：由互联端系统的分组交换设备和通信链路构成的网状网络。如：分组交换路由器、链路层交换机、通信链路(光纤、铜缆、无线电、激光链路)

接入网

接入网的用途是将主机连接到边缘路由器上

接入网分类：

  • 光纤到户FTTH
  • 数字用户线DSL
  • 同轴电缆
  • 无线接入
  • 企业和家庭网络

网络核心的两大功能

  • 路由：确定数据分组从源到目标所使用的路径（全局操作）

  • 转发：路由器或交换机将接收到的数据分组转发出去（即移动到该设备的某个输出接口）（本地操作）

物理层

物理层的作用

物理层是计算机网络的最底层，主要作用是为数据链路层提供透明的、可靠的比特传输服务。它定义了网络设备如何通过各种物理媒体（如电缆、光纤、无线电波等）传输比特流，涉及信号的物理特性、传输媒介的规范、以及接口的标准化。
具体来说：

  1. 传输比特流：将上层数据封装为电信号、光信号或无线电波，并在物理媒介上传输。
  2. 媒介的规范和接口：规定物理连接的属性（如插口形状、信号电平、电缆类型等）。
  3. 同步和复用：支持信号同步（时钟）、复用技术（如时分复用、频分复用）。
  4. 错误检测：通过硬件机制检测物理层的信号损坏和传输错误。

简而言之，物理层的目标是确保设备间比特流能够正确传输，并在各种物理媒介上保持可操作性和一致性。

物理层设备

  1. 中继器（Repeater）
    • 主要功能：再生和放大信号，以延长信号传输距离。(由于存在损耗, 在线路上传输的信号功率会逐渐衰减,衰减到一定程度时将造成信号失真,因此会导致接收错误。中继器就是为解决这一问题而设计的。它完成物理线路的连接,对衰减的信号进行放大,保持与原数据相同)
    • 特点：
      • 只能连接两个同一协议的网络段。
      • 不隔离冲突域，不隔离广播域。
      • 遵循 5-4-3规则：在10BASE-5以太网中，一个网络最多可以使用5个网络段连接，其中最多4个中继器串联，且只有3个段可用于主机连接，其余2个为中继器间的连接。
  2. 集线器（Hub）
    • 主要功能：多端口中继器，负责接收来自一个端口的信号并广播到所有其他端口。
    • 特点：
      • 工作在半双工模式下，所有端口共享一个冲突域。
      • 无法隔离冲突域和广播域，网络规模较大时容易产生冲突和广播风暴。
      • 在现代网络中逐渐被交换机取代。

信道的基本概念

信道是物理层传输信息的媒介，是一条通信线路中用于特定方向信息传递的部分。通信信道通常包括以下类型：

  1. 单工通信信道（Simplex Channel）
    • 特点：只能实现单向通信，即一方只能发送，另一方只能接收。
    • 示例：广播电视的信号传输。
  2. 半双工通信信道（Half-Duplex Channel）
    • 特点：双方可以交替发送和接收数据，但同一时刻只能完成一个方向的通信。
    • 示例：对讲机通信。
  3. 全双工通信信道（Full-Duplex Channel）
    • 特点：双方可以同时进行数据的发送和接收，提高通信效率。
    • 示例：电话通信、现代以太网通信。

物理层常见传输介质

  1. 导向介质
    • 双绞线：价格低廉，传输距离有限，适合局域网。
    • 同轴电缆：抗干扰能力强，适合较远距离。
    • 光纤：传输速率高，信号衰减小，适合远距离和高速网络。
  2. 非导向介质
    • 无线电波：适用于无线局域网（Wi-Fi）。
    • 微波：适用于远距离传输，如卫星通信。
    • 红外线：短距离设备间通信，如遥控器。

信道复用

概念：
复用（Multiplexing）是一种通信技术，允许多个用户共享同一信道进行通信，从而降低成本、提高信道利用率。

  1. 频分复用（FDM）

    • 将整个信道的频带分为多个子频带，每个用户在通信过程中占用一个固定的频带资源。
    • 特点：所有用户在同一时间占用不同的频率带宽资源。

  2. 时分复用（TDM）

    • 将时间划分为一段段等长的时分复用帧（TDM帧），每个用户在帧中占用固定的时隙。
    • 特点：所有用户在不同时间占用相同的带宽资源。

  3. 波分复用（WDM）

    • 光的频分复用，通过一根光纤同时传输多个光载波信号。

  4. 码分复用（CDM）

    • 各用户使用经过特殊挑选的不同码型，因此彼此之间不会产生干扰。

数据链路层

数据链路层在物理层提供的服务的基础上向网络层提供服务，其最基本的服务是将源自网络层来的数据可靠地传输到相邻节点的目标机网络层。数据链路层在不可靠的物理介质上提供可靠的传输。

数据链路层的作用

  1. 物理地址寻址：使用 MAC 地址来标识网络中的设备，保证数据帧能够准确到达目的节点。
  2. 数据成帧：将网络层传输的数据（数据包）封装为帧，并在帧头和帧尾添加必要的控制信息。
  3. 流量控制：防止发送方过快地发送数据，导致接收方来不及处理。
  4. 差错控制：检测并修正数据传输中的错误（如奇偶校验、CRC）。
  5. 透明传输：通过转义字符技术，实现对特殊控制字符的正确处理，使帧内容不受干扰。
  6. 重传机制：当帧未能正确传输时，启动自动重传请求（ARQ），提高数据传输的可靠性。

封装成帧

帧是数据链路层的基本传输单位，包含以下部分：

  1. 帧头：包含目的地址、源地址和其他控制信息，用于标识和管理帧传输。
  2. 数据字段：封装上层协议数据包，通常长度为 46 至 1500 字节（以太网）。
  3. 帧尾：通常用于差错检测（如 CRC）。

透明传输

透明传输的核心思想是：

  • 使帧数据中的控制字符（如帧边界标识符）被视为普通数据而不影响传输。
  • 实现方法：在控制字符前插入转义字符 ESC，接收方通过识别 ESC 来正确处理数据。

差错检测

数据链路层通过以下机制检测并处理传输中的错误：

  1. 奇偶校验码
    • 在数据末尾附加一个比特位，使整个数据的 1 的个数为奇数或偶数。
    • 局限性：无法检测偶数位出错。
  2. 循环冗余校验码（CRC）
    • 使用多项式生成校验码，能有效检测多位错误。
    • 被广泛应用于以太网帧的错误检测。

最大传输单元MTU

最大传输单元MTU(Maximum Transmission Unit)，数据链路层的数据帧不是无限大的，数据帧长度受MTU限制.

路径MTU：由链路中MTU的最小值决定。

以太网协议详解

MAC地址：每一个设备都拥有唯一的MAC地址，共48位，使用十六进制表示。

以太网协议：是一种使用广泛的局域网技术，是一种应用于数据链路层的协议，使用以太网可以完成相邻设备的数据帧传输：

以太网帧结构：

  • 类型：标识上层协议（2字节）
  • 目的地址和源地址：MAC地址（每个6字节）
  • 数据：封装的上层协议的分组（46~1500字节）
  • CRC：循环冗余码（4字节）
  • 以太网最短帧：以太网帧最短64字节；以太网帧除了数据部分18字节；数据最短46字节；

MAC地址（物理地址、局域网地址）

  • MAC地址长度为6字节，48位；
  • MAC地址具有唯一性，每个网络适配器对应一个MAC地址；
  • 通常采用十六进制表示法，每个字节表示一个十六进制数，用 - 或 : 连接起来；
  • MAC广播地址：FF-FF-FF-FF-FF-FF。
